# 開発方針・コーディング規約

これ以降のコード生成では、以下のルールを厳守してHTMLファイルを作成してください。

## 1. ファイル構成
- HTML, CSS, JavaScript はすべて1つの `.html` ファイル内に記述する（外部ファイルに分割しない）。
- コピペ1回でVercel等にデプロイできる「ポータビリティの高さ」を最優先する。

## 2. デザイン・UI（CSS）
- スマホでの閲覧を前提としたレスポンシブデザインにする（`meta viewport` 必須）。
- Flexboxを活用し、画面の中央（または上部）に見やすく配置する。
- 全体的にモダンでクリーンなUIにする。
  - 背景色: 薄いグレー（例: `#f0f0f0`）
  - メイン要素: 白背景、角丸（`border-radius`）、軽い影（`box-shadow`）のカード型デザイン
  - フォント: OS標準のゴシック体（`sans-serif`）

## 3. ロジック（JavaScript）
- Reactなどのフレームワークは使わず、Vanilla JS（素のJavaScript）で記述する。
- 履歴やリストを追加する場合は、常に「最新のデータが一番上」に来るように配慮する（`unshift` の活用など）。

## 4. SEO・シェア機能
- 検索エンジン用の `<meta name="description">` を必ず含める。
- 𝕏（旧Twitter）などでシェアされた際に見栄えが良くなるよう、OGPタグ（`og:title`, `og:image`, `twitter:card` など）を標準装備する。
